Overview

Through this project, we simulated a physical cloth by creating a grid of point masses and connecting point masses to each other with different kinds of springs. We then simulated the behavior of objects through numerical integration; for each point mass, we took into account external forces as well as the spring forces point masses apply on each other to come to a net force on each point mass. With the net force of each point mass, we could simulate the movement of each point mass through kinematics equations. We then simulated the collisions of point masses with planes and spheres, by checking if each point mass had crossed a plane or the surface of a sphere. If it had, we rectified each point by finding the tangent to the plane or sphere and then moving the point back across the boundary. Then, we simulated self collisions for point masses. We achieved this through hashing point masses, where the hash function would hash closer point masses together. We achieved this by splitting our world into 3d boxes and truncating point mass positions into the closest 3d box. Thus, when iterating over point masses to check for collisions, we only needed to check other points within the same sub-box of the world, greatly reducing computational cost. In the case of collisions, we would compute correctional vectors and average them across all collided points for some given point mass. Finally, we implemented various shaders, such as diffuse, blinn-phong, mirror. We also added textures to our surfaces. We also implemented bump and displacement mapping, allowing us to add height to our surfaces.

Part 5

  • Explain in your own words what is a shader program and how vertex and fragment shaders work together to create lighting and material effects.

A shader program is a program written in a special language for the computer's graphics card. It runs in parallel and renders the scene. Vertex and fragment shaders are two main types. The vertex shader deals with the shape and position of objects; it moves the corners of triangles around and sets them up for drawing. The fragment shader takes the output of vertex shaders and adds color and texture to each pixel. It figures out what color each little pixel (or fragment) on the screen should be based on lighting, textures, and other effects.

  • Explain the Blinn-Phong shading model in your own words. Show a screenshot of your Blinn-Phong shader outputting only the ambient component, a screen shot only outputting the diffuse component, a screen shot only outputting the specular component, and one using the entire Blinn-Phong model.

Blinn-Phong shading is an algorithm that mimics how surfaces react to light. It has three main components: ambient, diffuse, and specular lighting. Ambient lighting provides a base level of illumination even when there's no direct light source hitting a surface. Diffuse lighting determines how much light is scattered and reflected off a surface in all directions, creating the basic color and brightness. Specular lighting models shiny highlights that appear on smooth surfaces when light reflects off them, depending on the angle between the light source, viewer, and surface.
